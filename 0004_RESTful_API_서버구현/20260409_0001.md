



#### JSON-Web-Token(JWT)
```
JWT 문자열은 JSON 객체를 인코딩해서 구성되는데,
완성된 문자열은 헤더(header)-내용(payload)-서명(signature)로 구성된다.
각 부분은 중간에 "." 를 통해서 구분된다.


Header
JWT 의 유형 및 해싱 알고리즘을 지정한다.
주로 이 정보는 Base64 로 인코딩 되어 있다.

Payload
토큰에 포함되는 claim 정보가 담겨 있다.
클레임은 사용자에 대한 정보 또는 기타 데이터를 나타내며, 세가지 유형으로 나뉜다.
등록된(registered), 공개(public), 비공개(private).
이 부분도 Base64 로 인코딩 된다.

Signature 
Header 와 Payload 를 함께 사용해 생성된 서명이다.
이 서명은 토큰이 변조되지 않았음을 보장한다.
이 서명은 특정한 암호화 알고리즘에 의해 생성된다.


JWT 문자열읜 jwt.io 사이트에서 어떤 구조인지 파악할 수 있다.
```
