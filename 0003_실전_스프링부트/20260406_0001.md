#### 스프링부트 애플리케이션 종료 
```aiignore
server.shutdown=graceful
server.lifecycle.timeout-per-shutdown-phase=1m

server.shutdown 기본값은 immediate 이다.
아무런 설정을 하지 않으면 어플리케이션은 즉시 종료된다.

안전종료를 설정하면 처리중인 요청이 완료될 때 까지 기다려주는 타임아웃을 설정 할 수 있다.
이 타임아웃은 server.lifecycle.timeout-per-shutdown-phase 로 지정할 수 있으며 기본값은 30s 이다.
```

#### 어플리케이션 설정 관리(@PropertySource 사용)
```aiignore
package com.colamanlabs.springboot_test_2001;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.PropertySource;
import org.springframework.core.env.Environment;

@Configuration
@PropertySource("classpath:dbcConfig.properties")
public class DBConfiguration
{
@Autowired
private Environment env;

    @Override
    public String toString()
    {
        String user = env.getProperty("user");
        String pwd = env.getProperty("password");
        return String.format("user: %s, pwd: %s", user, pwd);
    }
}
```

#### 어팬더와 로거
```aiignore
logger 
로거는 한개 이상의 appender 를 사용해서 로그 메세지 표시를 담당하는 로깅 프레임워크의 컴포넌트이다.
필요에 따라 다양한 로그 수준을 가진 다수의 로거를 정의할 수 있다.

appender
appender 를 사용해서 로그가 출력되는 대상과 로깅 포맷을 지정할 수 있다.
로그 메세지가 출려고디는 매체에 따라, 다양한 어펜더가 있다.
콘솔 어펜더는 애플리케이션의 콘솔에 로그를 출력하고,
파일 어팬더는 로그메지지를 파일에 출력한다.

RollingFileAppender 는 시간과 날짜 기반으로 별도의 파일에 로그를 출력한다.

```



#### 스프링부트 개발자 도구 
```aiignore
1. 프로퍼티 기본값 
- 변경사항을 바로 확인할 수 있도록, 기본적으로 캐시기능을 모두 비활성화 한다.

2. 자동재시작
- 클래스패스에 변경사항이 있을 때마다 자동으로 어플리케이션을 재시작 해준다. 
```


#### 스프링부트 액추에이터 
```aiignore
스프링부트 액추에이터(actuator)는 스프링부트 애플리케이션 모니터링과 관리에 필요한 기능을 제공한다.
액추에이터의 주요장점중 하나는 애플리케이션 운영에 필요한 광범위한 기능을 직접 구현할 필요 없이 쉽게 사용할 수 있다는 점이다. 
```
