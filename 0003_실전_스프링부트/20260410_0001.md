


```
토크기반 인증방식의 가장 큰 문제는 토큰이 탈취되는 상황인데,
JWT 는 내용(payload)에 만료시간(exp)을 지정할 수 있어서 탈취된 토큰의 피해를 줄일 수 있다.
```


#### 
```aiignore
JWT 를 이용하는 경우 사용자가 로그인을 하면 응답 메세지로 AccessToken 과 RefreshToken 을 발행해 준다.
AccessToken 은 사용자가 API 서버에서 원하는 자원에 요청(request)할 때 같이 전송한다.

API 서버에서는 시큐리티의 필터를 이용해서 해당 사용자가 정상적인 Access Token 을 가지고 있는지를 확인하고 원하는 데이터를 제공한다.
Access Token 은 매번 API 서버와 호출되는 서비스 사이에서 오고 가기 때문에 탈취의 가능성이 높다.

때문에 Access TOken 의 만료시간은 상당히 짧게 지정하낟.
예를 들어 만료시간을 10분으로 지정하면 토큰이 탈취되더라도 10분이 지나면 사용할 수 없기 때문에 피해를 줄일 수 있다.

일반적으로는 1일 이내의 시간으로 Access Token 의 만료시간을 지정하는 경우가 많다.

Access Token 의 만료시간을 짧게 지정했을 때 단점은 API 서버를 호출하는 쪽에서 10분마다 다시 Access Token 을 발급받아야만 하는 상황이 생긴다는 점이다.
이를 보안하기 위해 Refersh Token 을 지정하낟.

Refresh Token 은 Access Token 보다는 상당히 긴 시간의 만료시간을 가지는 JWT 이다. 

사용자는 Access Token 이 만료되었지만, Refresh Token 은 아직 만료되지 않았다면
Refresh Token 을 이용해서 새로운 Access Token 을 요청하게 한다. 

API 서버에서는 Refresh Token 이 만료되지 않았다면,
새로운 Access Token 을 전송해주고,
상황에 따라서 새로운 Refresh Token 을 같이 발행해서 보낸다.

Access Token 과 달리 Refresh Token 은 토큰을 갱신할 때만 전송하는데,
이렇게 하면 Refresh Token 의 노출 횟수를 줄여서 조금 더 안전하게 만들 수 있다.
```


#### Access Token 과 Refresh Token 이 모두 탈취당한다면
```aiignore
Refresh Token 이 탈취된다면 공격자는 이를 이용해 새로운 액세스 토큰을 발급받아 사용자 계정에 대한 장기적인 접근권한을 가질 수 있다.
이는 보안상 큰 위협이 된다.
따라서, Refresh Token 이 탈취되는 상황에 대비해 다음과 같은 보안조치를 고려할 수 있다.

보안조치

- 짧은 수명의 Refresh Token 사용
Refresh Token 의 수명을 짧게 설정하여, 탈취된 Refresh Token 의 유효기간을 최소화 한다.
이를 통해 공격자가 Token 을 사용하는 시간을 제한할 수 있다.

- 기기와 세션에 대한 고유한 식별자 사용
각 기기와 세션에 대해 고유한 식별자를 사용하여 Refresh Token 을 발급한다.
이렇게 하면 특정 기기나 세션에만 Refresh Token 을 사용할 수 있다.

- 토큰의 사용 위치 제한
Refresh Token 이 사용될 수 있는 IP 주소나 지리적 위치를 제한한다.
이를 통해 의심스러운 위치에서의 토큰 사용을 방지할 수 있다.

- 이상활동 모니터링 및 알람
Refresh Token 사용 패턴을 모니터링 하여 이상활동이 감지되면 사용자에게 알림을 보내고,
추가 인증을 요구하거나, 해당 토큰을 무효화 한다.

- Refresh Token 무효화
사용자 로그아웃시 또는 특정 보안 이벤트 발생시 Refresh Token 을 즉시 무효화 한다.
사용자가 비밀번호를 변경하거나 의심스러운 활동을 보고하면 Refresh Token 을 무효화 한다.

- Multi-Factor Authentication(MFA) 
중요한 작업이나 민감한 정보에 접근할 때는 Multi-Factor Authentication 을 요구하여 추가적인 보안 레이어를 제공한다.

- Secure Storage
Refresh Token 을 안전하게 저장한다.
웹 애플리케이션 에서는 Secure Cookie 를 사용하고, 
모바일 애플리케이션에서는 안전한 저장소(Secure Storage) 를 사용한다.

- 토큰 회전(Token Rotation)
Refresh Token 을 사용하여 새로운 Access Token 을 발급받을 때마다, 
새로운 Refresh Token 도 함께 발급하고,
이전 Refresh Token 을 무효화한다.
이를 통해 탈취된 Refresh Token 의 재사용을 방지할 수 있다.
```
